title: Suppressing type errors
description: Learn how to suppress type errors with comments and decorators

Rules can be ignored in specific locations in your code (instead of disabling the rule entirely) to silence false positives or permissible violations.

<Note>
To disable a rule entirely, set it to the `ignore` level as described in the [rules documentation](/reference/rules).
</Note>

## ty suppression comments

To suppress a rule violation inline add a `# ty: ignore[<rule>]` comment at the end of the line:

```python
a = 10 + "test"  # ty: ignore[unsupported-operator]
```

### Multi-line violations

Rule violations spanning multiple lines can be suppressed by adding the comment at the end of the violation's first or last line:

<CodeGroup>
```python First line
def sum_three_numbers(a: int, b: int, c: int) -> int: ...

sum_three_numbers(  # ty: ignore[missing-argument]
    3,
    2
)
```

```python Last line
def sum_three_numbers(a: int, b: int, c: int) -> int: ...

sum_three_numbers(
    3,
    2
)  # ty: ignore[missing-argument]
```
</CodeGroup>

### Multiple violations

To suppress multiple violations on a single line, enumerate each rule separated by a comma:

```python
sum_three_numbers("one", 5)  # ty: ignore[missing-argument, invalid-argument-type]
```

<Warning>
Enumerating rule names (e.g., `[rule1, rule2]`) is optional. However, we strongly recommend suppressing specific rules to avoid accidental suppression of other errors.
</Warning>

## Standard suppression comments

ty supports the standard [`type: ignore`](https://typing.python.org/en/latest/spec/directives.html#type-ignore-comments) comment format introduced by PEP 484.

ty handles these similarly to `ty: ignore` comments, but suppresses all violations on that line, even when `type: ignore[code]` is used.

```python
# Ignore all typing errors on the next line
sum_three_numbers("one", 5)  # type: ignore
```

<Accordion title="When to use type: ignore vs ty: ignore">
  - Use `# ty: ignore[rule-name]` for ty-specific suppressions with precise rule targeting
  - Use `# type: ignore` for broad suppression compatible with other type checkers like mypy
  - Note that `# type: ignore[code]` in ty suppresses all violations, not just the specified code
</Accordion>

## Multiple suppression comments

To suppress a typing error on a line that already has a suppression comment from another tool, add the `# ty: ignore` comment to the same line.

For example, to suppress a type error and disable formatting for a specific line:

<CodeGroup>
```python With fmt: skip
result = calculate()  # ty: ignore[invalid-argument-type]  # fmt: skip
```

```python With fmt: off
result = calculate()  # fmt: off  # ty: ignore[invalid-argument-type]
```

```python With noqa
result = calculate()  # ty: ignore[invalid-argument-type]  # noqa: E501
```
</CodeGroup>

## Unused suppression comments

If the `unused-ignore-comment` rule is enabled (see the [rules reference](/reference/rules)), ty will report unused `ty: ignore` and `type: ignore` comments.

`unused-ignore-comment` violations can only be suppressed using `# ty: ignore[unused-ignore-comment]`. They cannot be suppressed using `# ty: ignore` without a rule code or `# type: ignore`.

```python
# This will trigger unused-ignore-comment if the error is fixed
a = 10 + 20  # ty: ignore[unsupported-operator]

# Suppress the unused-ignore-comment warning
a = 10 + 20  # ty: ignore[unsupported-operator]  # ty: ignore[unused-ignore-comment]
```

## `@no_type_check` decorator

ty supports the [`@no_type_check`](https://typing.python.org/en/latest/spec/directives.html#no-type-check) decorator to suppress all violations inside a function.

```python
from typing import no_type_check

def sum_three_numbers(a: int, b: int, c: int) -> int:
    return a + b + c

@no_type_check
def main():
    sum_three_numbers(1, 2)  # no error for the missing argument
```

<Note>
Decorating a class with `@no_type_check` isn't supported.
</Note>

## Best practices

<Steps>
  <Step title="Always specify rule names">
    Use `# ty: ignore[rule-name]` instead of bare `# ty: ignore` to avoid suppressing unintended errors.
    
    ```python
    # Good
    result = foo()  # ty: ignore[invalid-argument-type]
    
    # Bad - suppresses all errors
    result = foo()  # ty: ignore
    ```
  </Step>
  
  <Step title="Add explanatory comments">
    Document why the suppression is necessary.
    
    ```python
    # Third-party library has incomplete type stubs
    data = parse_json(raw)  # ty: ignore[invalid-argument-type]
    ```
  </Step>
  
  <Step title="Enable unused-ignore-comment">
    Keep your suppressions up to date by enabling the `unused-ignore-comment` rule (see the [rules reference](/reference/rules)).
  </Step>
  
  <Step title="Prefer fixing over suppressing">
    Only suppress errors when fixing them isn't practical or when dealing with legitimate limitations in the type system.
  </Step>
</Steps>
