Get up and running with ty in just a few minutes.

## Prerequisites

Before you start, make sure you have:

- Python 3.8 or later installed
- A Python project you want to type check

<Info>
  If you don't have Python installed, download it from [python.org](https://www.python.org/downloads/) or install it via your system's package manager.
</Info>

## Run ty with uvx

The fastest way to try ty is with `uvx` (no installation required):

<Steps>
  <Step title="Run ty check">
    Navigate to your Python project directory and run:
    
    ```bash
    uvx ty check
    ```
    
    ty will automatically:
    - Discover your Python environment
    - Find all Python files in your project
    - Check them for type errors
    
    <Note>
      If you're in a project with dependencies, you may need to use `uv run` or activate your virtual environment first for ty to find your dependencies.
    </Note>
  </Step>
  
  <Step title="Review the results">
    ty will display any type errors it finds with detailed diagnostics:
    
    ```
    error[invalid-argument-type]: Expected type `int`, but got `str`
      |
    5 | result = add_numbers("5", 10)
      |                      ^^^
      |
    ```
    
    Each diagnostic includes:
    - The error type and rule name
    - The specific location in your code
    - A clear explanation of what's wrong
  </Step>
  
  <Step title="Check specific files">
    You can also check specific files instead of your entire project:
    
    ```bash
    uvx ty check path/to/file.py
    ```
    
    Or multiple files and directories:
    
    ```bash
    uvx ty check src/ tests/test_main.py
    ```
  </Step>
</Steps>

## Environment discovery

ty needs to discover your installed packages to check your use of imported dependencies.

ty will automatically find packages in:
- The active virtual environment (via `VIRTUAL_ENV`)
- A `.venv` directory in your project root or working directory
- The `python3` or `python` binary in your `PATH`

You can also explicitly specify a Python interpreter:

```bash
uvx ty check --python /path/to/python
```

<Tip>
  See the [module discovery](/concepts/modules) documentation for more details on how ty finds your dependencies.
</Tip>

## Watch mode

Run ty in watch mode to automatically recheck files as you edit them:

```bash
uvx ty check --watch
```

ty uses fine-grained incremental analysis to make subsequent checks much faster than running `ty check` repeatedly.

## Configure rules

ty's type checking diagnostics are configurable. You can adjust rule severity levels in a `pyproject.toml` or `ty.toml` file:

```toml pyproject.toml
[tool.ty]
# Treat all warnings as errors
error-on-warning = true

# Configure specific rules
[tool.ty.rules]
invalid-argument-type = "error"
unknown-member-access = "warn"
```

See the [configuration guide](/guides/configuration) for all available options.

## Suppress specific errors

You can suppress specific type errors using inline comments:

```python
# ty: ignore[invalid-argument-type]
result = add_numbers("5", 10)
```

Learn more about suppression in the [suppression guide](/guides/suppression).

## Configure file selection

Persistently configure which files to check in your configuration file:

```toml pyproject.toml
[tool.ty]
include = ["src/**/*.py"]
exclude = ["tests/**"]
```

See the [file exclusions](/concepts/file-exclusions) documentation for more details.
